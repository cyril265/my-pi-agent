description: Review changes for bugs, risk, and maintainability
---
Treat this as a review task.

Workflow:
1. Inspect the relevant diff or changed files.
2. Use `reviewer` as the primary specialist.
3. If additional local context is needed, use `search` to understand surrounding code.
4. Prioritize correctness, regressions, maintainability, and unnecessary complexity.

Output guidance:
- Organize findings by severity.
- Include exact files and lines when possible.
- Prefer actionable findings over generic commentary.
- If the changes look good, say so clearly and briefly.
